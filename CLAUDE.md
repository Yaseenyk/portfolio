# CLAUDE.md

Portfolio + AI systems lab for Yaseen Khatib (Senior Full-Stack AI Engineer).
Next.js 14 App Router, **static export** (`output: "export"`), strict
TypeScript, Tailwind, Framer Motion. Deployed to GitHub Pages at
https://yaseenkhatib.streamerosai.com (custom domain, no basePath; DNS on
GoDaddy — Cloudflare is compute-only).

## Commands

```bash
npm run dev      # dev server :3000
npm run build    # next build + scripts/generate-llms-full.mjs → ./out
npm run lint
```

Verify changes against the built output in `out/` (grep the HTML) — metadata,
JSON-LD, and OG tags only exist post-build.

## Architecture — the parts that bite

- **Content is code.** Blog posts are typed `BlogPost` objects in
  `src/content/posts/*.tsx`, registered in `src/lib/blog.tsx` (`POSTS` array,
  newest first). AI-written posts are `.mdx` in `src/content/blog/` (merged
  via `src/lib/mdx.ts`; TSX wins slug collisions).
- **Never import `lib/blog` from a client component** — it pulls every post
  body into the bundle. Server pages map to plain props (see `app/page.tsx`
  → `FoundersLog`/`RecentPosts`).
- **SEO layer**: sitewide entity `@graph` (`lib/seo.ts`, `PERSON_ID`) emitted
  in the root layout; pages reference `personRef` by `@id` — never redeclare
  Person. All JSON-LD goes through `components/JsonLd.tsx`. Breadcrumbs via
  `breadcrumbJsonLd()`. Tag pages generate from `lib/tags.ts`.
- **OG covers**: `public/og/<slug>.jpg` is auto-attached (meta + page hero)
  by `app/blog/[slug]/page.tsx`. Generated by `scripts/generate-og-images.py`
  (one-time) and `scripts/ai_writer.py` (per new post). Style contract is
  brand-locked: flat vector on ink #05070A, cyan/violet only, NO glows or
  warm hues — keep both scripts' IMAGE_STYLE_RULES in sync.
- **Headings**: solid near-white; gradient (`GradientText`) is for 1–2 accent
  words only, never a full title (washes out, hurts readability).
- **Framer + Tailwind transforms conflict**: motion components write inline
  `transform`, clobbering `-translate-*` classes — position with a plain
  wrapper div (see ArchitecturePipeline). Hero h1 must stay visible at first
  paint (no opacity-0 initial) — it's the LCP element. Never autofocus
  off-screen inputs on load (scroll-hijack killed LCP once).

## Autonomous pipelines (all push to main; expect bot commits to race yours)

- `ai_blogger.yml` — Mon/Wed/Fri 10:00 UTC: writes post + cover, commits.
  Voice contract in `ai_writer.py` bans LLM-tells; keep it strict.
- `deploy.yml` — Pages deploy on push and after AI Blogger; also pings
  IndexNow and passes `NEXT_PUBLIC_CONCIERGE_URL`.
- `worker.yml` — deploys `worker/` (Cloudflare: RAG `/api/chat` + MCP `/mcp`
  at yaseen-concierge.yaseenyk.workers.dev) and re-embeds the corpus into
  Vectorize after every Pages deploy. Corpus source: `/llms-full.txt`.
- Sibling repo `../linkedin-bot` posts a blog URL to LinkedIn daily at
  10:00 UTC (queue: `urls.txt`, state: `published.json`).

If `git push` is rejected, a bot committed — `git stash push package-lock.json`
(if dirty), `git pull --rebase`, push.

## Conventions

- Conventional commits (`feat(scope):`), push directly to `main` when done —
  deploys are push-triggered. PowerShell quoting: never put double quotes
  inside `git commit -m @'...'@` here-strings (splits args).
- Honest content only: no fabricated dates, testimonials, or metrics — real
  artifacts over claims is the site's whole thesis.
- Keys live in gitignored `.env` (OPENAI_API_KEY also in `../linkedin-bot/.env`).
